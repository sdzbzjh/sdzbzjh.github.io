PHP从入门到出家（1）——奇葩的大小写规则

发表于 2019-05-29 更新于 2021-05-12 分类于 吐槽PHP

大部分编程语言的符号是大小写敏感（case sensitive）的，少数（如Basic）是大小写不敏感（case insensitive）的。然而PHP两种都不是。

PHP的函数、方法以及关键字是大小写不敏感的，而变量、常量都是大小写敏感的。至于类，又是大小写不敏感的了。

WTF！

如果以大小写不一致的方式使用classloader，则在路径是大小写敏感的类UNIX环境下，就很容易产生问题。

尽管PHP大小写规则奇葩，但是PHP程序员并不在乎这个问题，绝大多数人还是遵循使用和manual规则一致的大小写的代码风格。

这导致一些很不友善的API命名，比如htmlspecialchars。

为什么不是html_special_chars或htmlSpecialChars？

了解函数是大小写不敏感后，我们就知道事实上htmlSpecialChars是可行的。实际上最初这个函数（在PHP2的文档中）就是以CamelCase方式命名的（HtmlSpecialChars）。但不知是出于什么原因，（或许是出于“统一”大小写风格？）PHP3开始，文档中所有函数全部小写了。

但是最初为什么函数被设计为大小写不敏感的？

PHP之父Rasmus在一个访谈中说：“…I then wrote a very simple parser that would pick tags out of HTML files and replace them with the output of the corresponding functions in the C library… The simple parser slowly grew to include conditional tags, then loop tags, functions, etc. At no point did I think I was writing a scripting language. I was simply adding a little bit of functionality to the macro replacement parser. I was still writing all my real business logic in C.”

其实最初的PHP也许比今天更接近一个模板引擎！早期PHP就是宏替换（macro replacement），不负责业务逻辑。

宏替换就是将HTML中的某些标签替换为C写的对应函数的输出。所以这个访谈里说，因为HTML的标签是大小写不敏感的的，所以在查找对应函数时，就按照大小写不敏感的方式了。后来的PHP发展的已经完全不同，但函数名就一直大小写不敏感了。

实际上不仅函数，那些早期就有的结构，比如if、for之类的关键字，还有echo、list之类并非是函数的特殊指令，也都是大小写不敏感的。

我们应该反过来问，为何在语言大部分元素都是大小写不敏感的情况下，变量名却是大小写敏感的？

下面我们讨论下class 。class的属性和常量是大小写敏感的，方法是大小写不敏感的。这样和普通变量、常量、函数的大小写情况正好是一致的。难道这么设计是为了一致性？

问题是类名本身为什么是大小写不敏感的？从直觉上说，类名难道不是更接近常量？

这是因为PHP的class一开始其构造器并非现在的__construct，而是和class同名的方法。因为方法是大小写不敏感的，所以class名也就是大小写不敏感的。多么天才的设计！

好了，现在是不是觉得自己可以很自豪的说，终于搞清楚PHP中各种大小写规则了！Naive！

PHP有goto语句，跳转到指定的标记。goto作为关键字，肯定是大小写不敏感的。可是标记是敏感的。
