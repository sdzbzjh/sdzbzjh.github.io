ES6从入门到放弃（4）——箭头函数

发表于 2019-05-27 更新于 2021-05-12 分类于 ECMAScript 6

简介

ES6标准新增了一种新的函数：Arrow Function（箭头函数），也称“胖箭头函数”，允许使用“箭头”=>定义函数，是一种简写的函数表达式。

必要性

传统的JavaScript函数语法不灵活，每次需要定义函数时，都必须输入function () {},这至少会出现两个问题：

  1. 代码输入快了容易输错成funciton或者functoin或者其它，=>不容易出错
  2. 节省大量代码

没有function、没有return,没有()，没有{},代码干净、简洁。

使用箭头函数不仅能够避免错误，还能精简代码。由于积累效应，每一部分少一点合起来就多了，能节省代码和大幅提高工作效率的场景。

语法

ES5

使用箭头函数，可以分两步实现同样的函数功能。

  1. 使用=>来替代关键词function。
  1. 函数体只有一条返回语句时，可以省略括号{}和return关键词。

如果只有一个参数时，()可省略。这是箭头函数最简洁的形式，常用于作用简单的处理函数

this

箭头函数内的this值继承自外围作用域。运行时它会首先到它的父作用域找，如果父作用域还是箭头函数，那么接着向上找，直到找到需要的this指向。

一道经典的关于this的面试题

ES5中的this特别容易出问题，指向非常混乱,ES6的箭头函数解决了这个问题。

箭头函数中的this其实是父级作用域中的this。箭头函数引用了父级的变量词法作用域就是一个变量的作用在定义的时候就已经被定义好，当在本作用域中找不到变量，就会一直向父作用域中查找，直到找到为止。

由于this在箭头函数中已经按照词法作用域绑定了，所以，用call或者apply调用箭头函数时，无法对this进行绑定，传入的第一个参数被忽略。

this已经在词法层面完成了绑定，通过call或apply方法调用一个函数时，只是传入了参数而已，对this并没有什么影响 。

特性

没有arguments

箭头函数不仅没有this，常用的arguments也没有。如果能获取到arguments，那它一定是来自父作用域的。

上例中如果箭头函数有arguments，就应该输出的是3，4而不是1，2。

箭头函数不绑定arguments，取而代之用rest参数…解决

箭头函数要实现类似纯函数的效果，必须剔除外部状态。箭头函数除了传入的参数之外，在普通函数里常见的this、arguments、caller统统没有！

如果在箭头函数引用了this、arguments或者参数之外的变量，那它们一定不是箭头函数本身包含的，而是从父级作用域继承的。

总结

优点

  • 箭头函数是使用=>语法的函数简写形式。这在语法上与C#、Java 8、Python（lambda函数）和 CoffeeScript的相关特性非常相似。

  • 非常简洁的语法，使用箭头函数比普通函数少些动词，如：function或return。

    1. () => { … } // 零个参数用 () 表示。
    2. x => { … } // 一个参数可以省略 ()。
    3. (x, y) => { … } // 多参数不能省略 ()。

  • 如果只有一个return，{}可以省略。

  • 更直观的作用域和this的绑定，更好的处理this的指向问题。

使用场景

箭头函数适合于无复杂逻辑或者无副作用的纯函数场景下，例如用在map、reduce、filter的回调函数定义

中，另外目前vue、react、node等库，都大量使用箭头函数，直接定义function的情况已经很少了。
