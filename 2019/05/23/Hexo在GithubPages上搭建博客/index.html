Hexo在GithubPages上搭建博客

Zeptinary
2019-05-23
随笔

Hexo

介绍

Hexo是一款快速、简介且高效的博客框架，具有如下优点：

  • Node.js 所带来的超快生成速度，让上百个页面在几秒内瞬间完成渲染。
  • Hexo 支持 GitHub Flavored Markdown 的所有功能，甚至可以整合 Octopress 的大多数插件。
  • 只需一条指令即可部署到 GitHub Pages, Heroku 或其他网站。
  • Hexo 拥有强大的插件系统，安装插件可以让 Hexo 支持 Jade, CoffeeScript。

Github Pages设置

介绍

GitHub Pages本用于介绍托管在GitHub的项目，不过，由于空间免费稳定，用来搭建博客非常不错。每个帐号只能有一个仓库来存放个人主页，而且仓库的名字必须是username/username.github.io，通过https://username.github.io 来访问。

注意，个人主页的网站内容是在master分支下的。

创建

Hexo部署到Github Pages

这是最关键的一步，把在本地web环境下预览到的博客部署到Github Pages上，然后就可以直接通过http://username.github.io访问了。

部署到github的原理

  1. 在Github上创建的那个特别的仓库（sdzbzjh/sdzbzjh.github.io）一个最大的特点就是其master中的html静态文件，可以通过链接http://sdzbzjh.github.io来直接访问。
  2. hexo -g生成一个静态网站（第一次会生成一个public目录），这个静态网站可以直接访问。
  3. 将hexo生成的静态网站，提交(git commit)到github上。
