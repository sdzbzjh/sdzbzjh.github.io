ES6从入门到放弃（1）——变量声明

Zeptinary
2019-05-24
ECMAScript 6

在ES6之前，变量声明只有var和function以及隐式声明三种，ES6则增加了let、const、import和class。

块级作用域

一个重要的概念，叫做“块级作用域”。顾名思义，块级作用域就是代码块（通常以大括号包裹）以及其内部访问受限的变量、常量和函数。

在ES6之前并没有块级作用域，只有函数作用域和全局作用域。

let

let声明的变量的作用域是块级作用域。

let非常适合用于for循环。使用var声明变量i时，没有绑定块级作用域，所以变量i为全局变量，每一次循环，新的i值都会覆盖旧的i值，结果最后输出的都是最后一轮i的值，即i的最终结果为10，实际上都是console.log(10)。

涉及的知识点：JS的事件循环机制，setTimeout的机制

用let声明的变量传入到for循环体的作用域后，不会发生改变，不受外界的影响。

暂时性死区（Temporal Dead Zone）

在一个块级作用域中，let声明的变量唯一属于这个块级作用域，不受外部变量的影响。

下面代码中，tmp = 'dream or bread'的赋值会报ReferenceError错误，因为在if块中用let声明了tmp，tmp就绑定了这个作用域，而let声明的临时死区导致了并不能在声明前使用，所以在声明前对变量赋值会报错。

let不具备var的变量提升的作用

暂时性死区的本质是：代码运行到当前作用域，所需要使用的变量就已经存在但不可获取，只有运行到声明变量的那一行代码，才可以获取和使用该变量。

暂时性死区的意义是标准化代码，将所有变量的声明放在作用域的最开始。

不允许重复声明

ES6之前使用var多次声明会造成变量覆盖而且不会报错，这就给调试增加了难度，而let能够直接扼杀这个问题在摇篮之中，因为会直接报错。

  1. 在相同的作用域内，用let声明变量时，只允许声明一遍，但var是可以多次声明的。
  1. 不能在函数内部重新声明参数

const

用于声明常量

const声明的变量是只读属性，声明时必须同时赋值，否则会报错，通常以大写字母命名。

这样做优点：

  1. 提高代码的可读性，阅读代码的人立刻会意识到不应该修改这个值
  2. 避免无意间修改变量值所导致的错误

比如使用Node.js的一些模块的时候，只是使用对应的模块（如http模块），但是并不需要修改Node.js的模块，这个时候就可以声明成const，增加了代码的可读性和避免错误。

支持块级作用域

const和let类似，也支持块级作用域.

不支持变量提升，有暂时性死区

const声明的常量也是不提升，同样存在暂时性死区，只能在声明的位置后面使用。

特殊情况

如果声明的是一个对象，那么对于对象本身是不允许重新赋值的，但是对于对象的属性是可以赋值的。

实际上const能保证的，并不是变量的值不得改动，而是变量指向的那个内存地址所保存的数据不得改动。

对于基本类型的数据（数值、字符串、布尔值），数据保存在栈中，变量指向的栈地址。

对于复合类型的数据（主要是对象和数组），数据保存在堆中，变量指向的栈地址，栈中保存的只是一个指向堆中数据的引用。

如果要彻底将对象冻结（不可修改其属性），应该使用Object.freeze(obj)方法。同理，数组也是一样的。

import

ES6采用import来代替CommonJS和AMD规范的require来导入模块。

$对象就是jquery中export暴露的对象。

如果想为输入的变量重新取一个名字，import命令要使用as关键字，将输入的变量重命名。

注意，import命令具有提升效果，会提升到整个模块的头部，首先执行。

class

ES6引入了类的概念，有了class这个关键字。类的实质还是函数对象。

constructor方法，就是构造方法，也就是ES5时代函数对象的主体，而this关键字则代表实例对象。

Class其实就是一个function，但是有一点不同，Class不存在变量提升，也就是说Class声明定义必须在使用之前。

总结

let与const相同点：

  • 块级作用域
  • 有暂时性死区
  • 约束了变量提升
  • 禁止重复声明变量

let与const不同点：

  • const声明的变量不能重新赋值
  • const变量声明时必须初始化
